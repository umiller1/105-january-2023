IDs vs Classes

As stated above, you'll want to use IDs when you need to apply unique or specific styles to an element that you don't want applied to anything else except for the element with the id on it. Think logos, main navigations, sidebar navs, footers, etc. Not all of these elements require IDs but sometimes you'll have multiple of the same element and you want one in particular to stand out or be different from the others.


An ID can only be used once per html file. Never duplicate IDs in an html file, always give other IDs different names.



Classes on the other hand are used when you want to apply the same styles to the same or different elements. Classes can be applied and duplicated as many times as you want. Think back to the .card classes we used in previous assignments to give sections and divs the same borders, padding, and margin!


Why use a class instead of a general HTML selector? Well simple! Here's an example: If you wanted to target every and any div to exist in your html, then you can use a general html selector. If you wanted to target every and any div that is inside of every and any section, then you can use a general html selector as well! However, if you only wanted to target every div that exist inside of your second section or your section with an id on it, then you can apply classes to each of those divs to target them!

Memorizing Div Placements

When we have a bunch of divs in a document, the best ways to remember them are:

  • Having good indentation.
    This allows us to see where everything is nested, including divs.
  • Applying classes and ids to divs that need them.
  • Using HTML comments to separate them or create line breaks that only appear in the html file and not to the end user.

Form Method: GET vs POST


The GET method is the default method for HTML forms and is used by the browser to ask the server to send back a given resource: "Hey server, I want to get this resource." In this case, the browser sends an empty body. Because the body is empty, if a form is sent using this method the data sent to the server is appended to the URL.


The POST method is a little different. It's the method the browser uses to talk to the server when asking for a response that takes into account the data provided in the body of the HTTP request: "Hey server, take a look at this data and send me back an appropriate result." If a form is sent using this method, the data is appended to the body of the HTTP request.
